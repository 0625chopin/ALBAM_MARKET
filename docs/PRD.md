# 알밤마켓 MVP PRD

> 출처: [`requirements/REQUIREMENTS.md`](./requirements/REQUIREMENTS.md) 기반으로 1인 개발 착수용 MVP를 압축한 문서입니다.
> 미결정/개선 항목은 [`ISSUES.md`](./ISSUES.md)를 참조하세요.

## 🎯 핵심 정보

**목적**: 판매자가 시작가로 경매를 열고 구매자가 입찰해 낙찰받은 뒤, 실시간 채팅으로 약속을 잡아 **직거래**하고 서로 평점을 남기는 C2C 경매 플랫폼.
**사용자**: 중고 물품을 경매로 합리적인 가격에 사고팔며 직접 만나 거래하려는 일반 개인 회원(판매자/구매자 권한 공동 보유).

## 🚶 사용자 여정

```
1. 홈(경매 목록) 페이지
   ↓ 상품 카드 클릭

2. 경매 상세 페이지 (현재가/남은 시간 확인)
   ↓ 입찰 또는 즉시구매 시도

   [비로그인] → 로그인 페이지 → (회원가입 페이지) → 다시 경매 상세 페이지
   [로그인]   → 입찰 / 즉시구매가 도달 시 즉시 낙찰
   ↓

3. 경매 종료 (36시간 만료 최고가 낙찰 / 입찰 없으면 유찰)
   ↓ 낙찰 확정

4. 거래 목록 페이지 (낙찰 건 표시)
   ↓ 분기

   [구매 진행] → 채팅 페이지(판매자와 약속) → 현장 직거래 → 거래완료(또는 자동완료)
   [낙찰 포기] → 경매취소 → 차순위 입찰자에게 그의 입찰가로 연쇄 이양 (입찰자 소진 시 유찰)
   ↓

5. 거래완료 후 → 상호 별점(10점) 부여 → 프로필 페이지의 레벨/평점 반영

6. [완료] → 홈으로 / 새 경매 등록(경매 등록 페이지)
```

## ⚡ 기능 명세

| **[F005]** | 즉시구매            | 즉시구매가 도달 시 경매 즉시 종료 및 해당 구매자 낙찰                                    | 빠른 거래 성사 경로               | 경매 상세 페이지                   |

| **[F007]** | 낙찰 포기(경매취소) | 낙찰자가 포기 시 차순위 입찰자에게 그의 입찰가로 연쇄 이양, 소진 시 유찰, 포기자 패널티  | 비구매 시 차선 구매자 보호        | 거래 목록 페이지                   |
| **[F008]** | 상품 내리기         | 입찰 전에는 자유롭게 내림, 입찰 후에는 제한(불가/패널티)                                 | 판매자 철회권 + 입찰자 보호       | 경매 상세 페이지, 거래 목록 페이지 |
| **[F009]** | 낙찰 후 실시간 채팅 | 낙찰 후에만 판매자-낙찰자 1:1 실시간 채팅(약속 조율), 메시지 저장                        | 직거래 약속의 유일한 소통 수단    | 채팅 페이지                        |
| **[F010]** | 거래완료            | 구매자가 거래완료 버튼 확정, 일정 시간 미클릭 시 자동완료                                | 거래 성사 확정 + 판매자 보호      | 거래 목록 페이지, 채팅 페이지      |
| **[F011]** | 상호 별점/평점      | 거래완료 후 판매자↔구매자 10점 별점 상호 부여(거래당 1회)                                | 신뢰 기반 거래의 평판 축적        | 거래 목록 페이지(평점 모달)        |
| **[F012]** | 평판/레벨 표시      | 판매/구매 역할별 평균 별점·레벨(거래 건수+별점) 노출                                     | 거래 상대 신뢰도 판단 근거        | 프로필 페이지, 경매 상세 페이지    |

- 관심목록(찜)
- 풀 마이페이지 대시보드(판매/구매 내역 통계, 패널티 현황)
- 관리자/신고·제재, 다국어

## ⚙️ 정책 상수 & 도메인 규칙

> 미결정 정책은 `lib/constants.ts`에 **임시 상수**로 관리하며, 확정 시 DB 정책값으로 이관합니다. 각 값에 관련 ISSUE 번호를 병기합니다.

| 상수/규칙           | 값                                    | 근거 상수 / 비고                                                       |
| ------------------- | ------------------------------------- | ---------------------------------------------------------------------- |
| 기본 경매 진행 시간 | **36시간**                            | `DEFAULT_AUCTION_DURATION_HOURS` (임시, ISSUE-001 — 추후 DB 이관)      |
| 최소 입찰 증가폭    | **1,000원**(정액)                     | `MIN_BID_INCREMENT` (임시, ISSUE-003 — 정액/정률/구간 정책 미확정)     |
| 자동완료 대기 시간  | **72시간**(3일)                       | `AUTO_COMPLETE_WAIT_HOURS` (임시, ISSUE-002 미확정)                    |
| 별점 범위           | **1~10점**                            | 거래당 각 1회(판매자↔구매자 상호)                                      |
| 중고등급(5종)       | new / like_new / good / fair / poor   | `PRODUCT_CONDITIONS` (라벨: 새상품(미개봉)·사용감 없음·적음·많음·고장) |
| 카테고리(10종)      | digital · appliance · furniture … etc | `CATEGORY_OPTIONS` (Mock 임시, Phase 4에서 `categories` 테이블로 이관) |
| 직거래 지역(17종)   | 시·도 단위(서울 … 제주)               | `REGION_OPTIONS`                                                       |
| 낙찰 포기 패널티    | 미확정                                | ISSUE-004 (점수/기준/제재 강도 미결정)                                 |
| 레벨 산정식         | 거래 건수 + 받은 별점 종합(역할별)    | ISSUE-005 (가중치/임계값 미확정)                                       |
| 입찰 후 상품 내리기 | 제한(불가 또는 패널티)                | ISSUE-006 (강도 미확정)                                                |

## 🔄 상태 전이

> 실제 코드 enum 기준 (`lib/types/product.ts`의 `ProductStatus`, `lib/types/transaction.ts`의 `TransactionStatus`). 한글 라벨은 `lib/constants.ts`의 `PRODUCT_STATUS_LABELS`/`TRANSACTION_STATUS_LABELS`.

### 상품(경매) 상태 — `ProductStatus`

```text
active(경매중) ─▶ won(낙찰)        # 36시간 만료 최고가 낙찰 또는 즉시구매가 도달
active ─▶ failed(유찰)             # 입찰자 없음
active ─▶ withdrawn(내림)          # 입찰 전 상품 내리기(입찰 후 제한)
won    ─▶ won(차순위 이양)         # 낙찰 포기 시 차순위 입찰자에게 그의 입찰가로 연쇄 이양
won(입찰자 소진) ─▶ failed(유찰)   # 연쇄 이양 대상 소진
won    ─▶ completed(완료)          # 거래완료/자동완료
```

| 값          | 라벨   |
| ----------- | ------ |
| `active`    | 경매중 |
| `won`       | 낙찰   |
| `failed`    | 유찰   |
| `withdrawn` | 내림   |
| `completed` | 완료   |

### 거래(transactions) 상태 — `TransactionStatus`

```text
pending(진행중) ─▶ completed(거래완료)       # 구매자 거래완료 클릭
pending ─▶ auto_completed(자동완료)          # 72시간 미클릭 시 자동 처리
pending ─▶ canceled(취소)                    # 낙찰 포기 → 차순위 이양 시 기존 거래 종료
```

| 값               | 라벨     |
| ---------------- | -------- |
| `pending`        | 진행중   |
| `completed`      | 거래완료 |
| `auto_completed` | 자동완료 |
| `canceled`       | 취소     |
