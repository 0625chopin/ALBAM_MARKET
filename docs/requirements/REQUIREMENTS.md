# 알밤마켓 요구사항 명세서 (REQUIREMENTS)

> C2C 온라인 경매 + 직거래 플랫폼 **"알밤마켓"**의 정식 요구사항 명세서입니다.
> 원본 요구사항은 [`REQUIREMENTS_O.md`](./REQUIREMENTS_O.md), 미결정/개선 항목은 [`../ISSUES.md`](../ISSUES.md)를 참조하세요.
> 작성 기준일: 2026-06-29

---

## 📌 1. 프로젝트 개요

| 항목        | 내용                                                                                         |
| ----------- | -------------------------------------------------------------------------------------------- |
| 서비스명    | 알밤마켓                                                                                     |
| 한줄 정의   | 판매자가 시작가를 정해 경매를 열고, 구매자가 입찰하여 낙찰 후 **직거래**하는 C2C 경매 플랫폼 |
| 거래 방식   | 낙찰자와 판매자가 **앱 내 실시간 채팅**으로 약속을 잡아 **현장 직거래**(온라인 결제 없음)    |
| 핵심 가치   | ① 경매를 통한 합리적 가격 형성 ② 실시간 소통 기반 안전한 직거래 ③ 상호 평판으로 신뢰 구축    |
| 회원 모델   | 단일 회원이 **판매자/구매자 권한을 공동 보유** (역할별 가입 분리 없음)                       |
| 기술 베이스 | Next.js(App Router) + Supabase(Auth/DB/Storage/Realtime), 기존 인증 스타터킷 확장            |

### 🎯 개발 접근 (CLAUDE.md 화면 우선 개발 준수)

- 모든 화면은 **컴포넌트 단위**로 구현하고 `https://localhost:3000/sample`에 배치한다.
- **Mock 데이터로 UI 먼저** 완성한 뒤 Supabase DB를 설계·연결한다.
- Mock 데이터와 실제 DB 데이터는 **동일한 TypeScript 타입**을 사용한다.
- 실제 데이터 전환 시 UI 컴포넌트는 수정하지 않고 **데이터 조회 부분만 교체**한다.

## 👤 2. 사용자 / 역할

| 역할          | 설명                                                                          |
| ------------- | ----------------------------------------------------------------------------- |
| 비회원(Guest) | 상품 목록/상세 열람, 검색/필터 가능. 입찰·채팅·등록 불가(로그인 유도)         |
| 회원(Member)  | 판매자/구매자 권한 공동 보유. 경매 등록·입찰·채팅·거래완료·평점·찜 모두 가능  |
| └ 판매자 역할 | 상품(경매) 등록, 상품 내리기, 거래 진행, 구매자에게 별점 부여                 |
| └ 구매자 역할 | 입찰, 낙찰, 경매취소(낙찰 포기), 거래완료 버튼, 판매자에게 별점 부여          |
| 관리자(Admin) | (향후) 신고/제재, 카테고리 관리, 패널티 정책 관리 — 본 MVP 범위 외, 확장 고려 |

## 🔑 3. 용어 정의

| 용어                | 정의                                                                               |
| ------------------- | ---------------------------------------------------------------------------------- |
| 시작가              | 판매자가 지정하는 경매 최저 입찰 기준 가격                                         |
| 즉시구매가(선택)    | 판매자가 설정 가능한 즉시 낙찰 가격. 입찰가가 도달하면 경매 즉시 종료(미설정 가능) |
| 현재가              | 현재까지의 최고 입찰가                                                             |
| 최소 입찰 증가폭    | 직전 최고가 대비 최소 인상 단위 (→ [ISSUE-003](../ISSUES.md))                      |
| 낙찰                | 경매 종료 시점의 최고 입찰자가 거래 권리를 갖는 것                                 |
| 유찰                | 입찰자가 없거나 모든 입찰자가 포기하여 거래가 성립되지 않은 상태                   |
| 경매취소(낙찰 포기) | 낙찰자가 구매를 포기하는 행위. 차순위 입찰자에게 그의 입찰가로 연쇄 이양됨         |
| 연쇄 이양           | 낙찰자 포기 시 차순위 → 그 다음 순위로 순차 이전, 입찰자 소진 시 유찰              |
| 패널티              | 낙찰 포기·정책 위반 시 부여되는 신뢰도 제재 (→ [ISSUE-004](../ISSUES.md))          |
| 거래완료            | 구매자가 직거래 완료를 확정하는 행위(또는 일정 시간 후 자동완료)                   |
| 레벨                | 거래 성사 건수 + 받은 별점으로 산정되는 회원 등급(판매자/구매자 역할별 보유)       |
| 기본 낙찰시간       | 경매 진행 시간. **상수 36시간** (→ [ISSUE-001](../ISSUES.md), 추후 DB 관리)        |

## 🗂️ 6. 데이터 모델 (개념 설계)

> 논리 설계 수준. 물리 스키마/RLS/타입 생성은 화면 우선 개발 완료 후 단계.
> Mock 데이터와 실제 DB는 동일 TypeScript 타입 사용.

| 엔티티           | 핵심 필드(요약)                                                                                                                                               |
| ---------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `profiles`       | id(=auth.users), nickname, region, avatar_url, seller_level, buyer_level, 별점 집계(판매/구매 각각), created_at                                               |
| `categories`     | id, name, slug                                                                                                                                                |
| `products`(경매) | id, seller_id, title, description, category_id, condition, region, start_price, buy_now_price?, current_price, status, auction_ends_at, winner_id, created_at |
| `product_images` | id, product_id, url, sort_order, is_primary                                                                                                                   |
| `bids`           | id, product_id, bidder_id, amount, status(active/won/abandoned), created_at                                                                                   |
| `transactions`   | id, product_id, seller_id, buyer_id, final_price, status(pending/completed/auto_completed/canceled), completed_at                                             |
| `ratings`        | id, transaction_id, rater_id, ratee_id, role(as_seller/as_buyer), score(1–10), comment, created_at                                                            |
| `chat_rooms`     | id, transaction_id(또는 product_id), seller_id, buyer_id, created_at                                                                                          |
| `messages`       | id, room_id, sender_id, content, created_at (Realtime 구독)                                                                                                   |
| `notifications`  | id, user_id, type, payload(jsonb), is_read, created_at                                                                                                        |
| `favorites`(찜)  | user_id, product_id, created_at (복합 PK)                                                                                                                     |
| `penalties`      | id, user_id, reason(abandon/withdraw/…), ref_id, created_at                                                                                                   |

**관계 요약**

- `profiles 1 ─ N products`(판매자) · `profiles 1 ─ N bids`(입찰자)
- `products 1 ─ N bids` · `products 1 ─ N product_images`
- `products 1 ─ 1 transactions`(낙찰 성립 시) · `transactions 1 ─ N ratings`(상호 최대 2건)
- `transactions 1 ─ 1 chat_rooms 1 ─ N messages`

## 🔄 7. 상태 전이도

### 7.1 상품(경매) 상태

```text
draft(작성) ─▶ active(경매중)
active ─▶ withdrawn(내림)        # 입찰 전에만 자유 / 입찰 후 제한
active ─▶ sold(낙찰)             # 36시간 만료 또는 즉시구매가 도달
active ─▶ ended_no_bid(유찰)     # 입찰자 없음
sold  ─▶ canceled_to_next(이양) # 낙찰자 포기 → 차순위로 이양(다시 sold)
sold(연쇄 이양 소진) ─▶ ended_no_bid(유찰)
sold ─▶ completed(거래완료)      # 거래완료/자동완료 시 transactions로 연결
```

### 7.2 거래(transactions) 상태

```text
pending(거래중) ─▶ completed(구매자 거래완료 클릭)
pending ─▶ auto_completed(미클릭 → 자동완료)   # 대기시간 ISSUE-002
pending ─▶ canceled(낙찰 포기 → 차순위 이양 시 기존 거래 종료)
```

| --------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Phase 0**           | (화면 우선) Mock 데이터 + 공용 타입 정의, 핵심 화면/컴포넌트 `/sample` 구현                                                                                   |
