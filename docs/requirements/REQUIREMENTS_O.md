# 알밤마켓 요구사항 원본 (REQUIREMENTS_O)

> 이 문서는 사용자가 전달한 **요구사항 원문**을 가공 없이 보존하기 위한 파일입니다.
> 분석·구조화된 정식 명세서는 [`REQUIREMENTS.md`](./REQUIREMENTS.md)를 참조하세요.

---

## 📥 원본 요구사항 (전달받은 그대로)

```text
알밤마켓

온라인 경매 플랫폼입니다.
판매자가 상품을 가격을 지정해 올리고, 구매자가 가격에 경매를 하는 시스템입니다.
상품은 판매자와 구매자가 대화를 하고 약속을 잡고 직거래 하는 방식 입니다.
판매자는 구매자에게, 구매자는 판매자에가 10점만점의 별점을 줄수 있습니다.
거래 성공시 구매자가 거래완료 버튼을 누를수 있습니다.
판매자의 거래가 성사 될수록 판매자의 레벨이 올라갑니다.
판매자가 팔고 싶지 않을때는 상품을 내릴수 있습니다.
구매자가 낙찰 후 사고 싶지 않을 경우 경매취소 버튼으로 이전 낙찰했던 사람에게 낙찰했던 가격으로 넘어갑니다.
회원은 판매자/구매자 공동 권한이 있습니다.
서로의 매신저 연락은 물건이 낙찰 후 할수 있습니다.
기본낙찰시간은 상수로 36시간으로 해주세요.(추후 DB 데이터로 관리 합니다.-> docs/ISSUES.md 파일에 등록해 주세요)
```

## ✅ 분석 과정에서 확인된 추가 결정사항 (Q&A 부록)

원본의 모호한 지점에 대해 2라운드의 확인 질문을 통해 아래와 같이 확정했습니다. 상세 반영 내용은 [`REQUIREMENTS.md`](./REQUIREMENTS.md)에 있습니다.

| #   | 항목                | 확정 내용                                                                                                           |
| --- | ------------------- | ------------------------------------------------------------------------------------------------------------------- |
| 1   | 경매 방식           | **오름차순(영국식) 경매 + 즉시구매가(선택)**. 시작가 이상 입찰, 36시간 후 최고가 낙찰. 즉시구매가 도달 시 즉시 종료 |
| 2   | 연락 수단           | **앱 내 실시간 채팅(WebSocket / Supabase Realtime)**. 낙찰 후 채팅방 자동 생성, 메시지 DB 저장                      |
| 3   | 거래완료 처리       | **구매자 거래완료 버튼 + 일정 시간 후 자동완료**(판매자 보호)                                                       |
| 4   | 평판 시스템         | **회원은 판매자 역할/구매자 역할 평점을 각각 보유. 레벨 = 성사 건수 + 별점(10점) 종합**                             |
| 5   | 낙찰 포기(경매취소) | **차순위 연쇄 이양 + 포기자 패널티**. 다음 입찰자 없으면 유찰                                                       |
| 6   | 상품 내리기         | **입찰 전 자유, 입찰 후 제한**(패널티/불가)                                                                         |
| 7   | 상품 정보           | 다중 이미지 업로드 · 카테고리 분류 · 직거래 지역 · 중고등급(상품 상태)                                              |
| 8   | MVP 부가기능        | 인앱 알림 · 검색/필터 · 관심목록(찜) · 마이페이지                                                                   |

**공통 전제**

- 직거래(현장 현금/계좌이체) 방식 → 플랫폼 내 온라인 결제 없음
- 회원은 판매자/구매자 권한을 공동 보유(역할 가입 분리 없음)
